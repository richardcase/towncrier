**Step 3: Run the C test with ASAN**

Compile the C test binary directly with ASAN flags to validate memory safety. This bypasses
the build.zig step to apply compiler-level sanitizers:

```bash
# Find the compiled object or recompile directly with clang + ASAN
clang -std=c11 -fsanitize=address,undefined \
  -I include/ \
  tests/c_abi_test.c \
  zig-out/lib/libtowncrier.a \
  -o /tmp/c_abi_test_asan

/tmp/c_abi_test_asan
```

Expected: exits 0, output contains `c_abi_test: PASS`, NO ASAN error output (no
`AddressSanitizer:` or `UndefinedBehaviorSanitizer:` lines in stderr).

If ASAN reports errors:
- `heap-use-after-free`: Check towncrier_free in c_api.zig — double-free or use after free.
- `stack-use-after-return`: A stub is returning a pointer to a local variable — fix it to return NULL.
- `undefined behavior`: Integer overflow or null dereference — fix the affected stub.

**Step 4: Verify all 13 ABI symbols are exported**

```bash
nm zig-out/lib/libtowncrier.a | grep towncrier_
```

Expected output must include ALL of the following symbol names (order does not matter):
- towncrier_init
- towncrier_free
- towncrier_start
- towncrier_stop
- towncrier_tick
- towncrier_add_account
- towncrier_remove_account
- towncrier_snapshot_get
- towncrier_snapshot_free
- towncrier_snapshot_count
- towncrier_snapshot_get_item
- towncrier_mark_read
- towncrier_mark_all_read

**Step 5: Verify no GTK or macOS-specific symbols leaked into the library**

```bash
nm zig-out/lib/libtowncrier.a | grep -i gtk || echo "PASS: no GTK symbols"
nm zig-out/lib/libtowncrier.a | grep -i "CoreFoundation\|NSObject\|objc_" || echo "PASS: no macOS symbols"
```

Expected: both commands print the PASS message (no matches).

**Step 6: Verify build.zig has the Linux platform guard**

```bash
grep "os.tag == .linux" build.zig
```

Expected: prints the line containing the platform guard.

Record all command outputs in the task summary for traceability.
