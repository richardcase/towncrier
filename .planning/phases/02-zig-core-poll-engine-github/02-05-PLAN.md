**MockServer struct** (bind to port 0 for OS-assigned ephemeral port, D-04 compliance):

The mock server must run in a separate thread and handle GET /notifications and PATCH /notifications/threads/:id.

Per-account state map key: the Authorization header value (Bearer {token}). Map value: the last `Last-Modified` value sent to that token. This enables stateful 304 simulation.

```zig
const MockServer = struct {
    server: std.net.Server,
    // Maps: "Bearer {token}" → last_modified string sent
    state: std.StringHashMap([]const u8),
    mutex: std.Thread.Mutex,
    thread: ?std.Thread,
    allocator: std.mem.Allocator,
    // Count PATCH /notifications/threads/* calls (for GH-04 verification)
    patch_count: std.atomic.Value(u32),
    // Count GET /notifications calls (for GH-03 verification)
    get_count: std.atomic.Value(u32),
    // Count 304 responses sent (for GH-03 verification)
    not_modified_count: std.atomic.Value(u32),

    pub fn init(allocator: std.mem.Allocator) !MockServer { ... }
    pub fn port(self: *MockServer) u16 { return self.server.listen_address.getPort(); }
    pub fn start(self: *MockServer) !void { ... } // spawn accept loop thread
    pub fn stop(self: *MockServer) void { ... }   // set shutdown flag, close server
};
```

**Mock response logic** (implement in the accept loop thread):

For each connection, parse the HTTP request line and headers:
- `GET /notifications` or `GET /notifications?participating=true`:
  - Read Authorization header value
  - Look up stored last_modified for this token in state map (under mutex)
  - Read If-Modified-Since header
  - If If-Modified-Since matches stored last_modified → respond 304 Not Modified (no body), increment not_modified_count
  - Else → respond 200 with fixture JSON + Last-Modified header + X-Poll-Interval: 60 header, update state map, increment get_count
- `PATCH /notifications/threads/*`:
  - Respond 205 Reset Content, increment patch_count
- Any other path → respond 404

```zig
    const deadline = std.time.milliTimestamp() + @as(i64, @intCast(timeout_ms));
```

For CORE-08 token check, implement:

```zig
fn assertTokenNotOnDisk(allocator: std.mem.Allocator, tokens: []const []const u8) !void {
    // Check the SQLite DB file path
    const db_dir = try std.fs.getAppDataDir(allocator, "towncrier");
    defer allocator.free(db_dir);
    const db_path = try std.fmt.allocPrint(allocator, "{s}/state.db", .{db_dir});
    defer allocator.free(db_path);
    const file = std.fs.openFileAbsolute(db_path, .{}) catch return; // DB may not exist yet
    defer file.close();
    const content = try file.readToEndAlloc(allocator, 10 * 1024 * 1024);
    defer allocator.free(content);
    for (tokens) |tok| {
        if (std.mem.indexOf(u8, content, tok) != null) {
            std.log.err("FAIL: token '{s}' found in DB file!", .{tok});
            std.process.exit(1);
        }
    }
}
```

**Exit behavior**: If any assertion fails, call `std.log.err("FAIL: {s}", .{message})` and `std.process.exit(1)`. On success, print `std.debug.print("ALL TESTS PASSED\n", .{})` and exit 0.

**Note on `std.http.Server` for MockServer**: Use `std.net.Address.listen()` to create a `std.net.Server`. Then in the accept thread, call `server.accept()` in a loop. For each accepted connection, read the request manually (first line + headers) and write the response. The Zig 0.14 HTTP server API is lower-level than a framework — you write raw HTTP/1.1 responses as strings.

If `std.net.Server.listen_address.getPort()` is not available in Zig 0.14 (RESEARCH.md Assumption A2), use `server.listen_address` directly (it is a `std.net.Address` union). For a TCP/IP address, use `server.listen_address.in.getPort()` or check the std.net.Address API. As a fallback, bind to a hardcoded loopback port (e.g., 14801) and use that.

The mock server accept loop must handle the shutdown case. Use a shutdown flag (std.atomic.Value(bool)) and close the listening socket to unblock `accept()`.
