Implement `pub fn open(db_path: []const u8) !sqlite.Db`:
- `sqlite.Db.init(.{ .mode = sqlite.Db.Mode{ .File = db_path }, .open_flags = .{ .write = true, .create = true }, .threading_mode = .MultiThread })`
- `try db.exec("PRAGMA journal_mode=WAL", .{}, .{});`
- `try db.exec("PRAGMA synchronous=NORMAL", .{}, .{});`
- `try migrate(&db);`
- `return db;`

Implement `pub fn migrate(db: *sqlite.Db) !void`:
- Try to read `SELECT version FROM schema_version LIMIT 1` — wrap in a catch to handle "no such table" error (table may not exist on first run)
- If version < 1 or table missing: execute the full schema_v1 DDL string, then `INSERT INTO schema_version VALUES (1)`
- schema_v1 DDL (use multiline string literal):
```sql
CREATE TABLE IF NOT EXISTS notifications (
    id            INTEGER PRIMARY KEY,
    account_id    INTEGER NOT NULL,
    api_id        TEXT    NOT NULL,
    service       INTEGER NOT NULL,
    notif_type    INTEGER NOT NULL,
    repo          TEXT    NOT NULL,
    title         TEXT    NOT NULL,
    url           TEXT    NOT NULL,
    updated_at    INTEGER NOT NULL,
    is_read       INTEGER NOT NULL DEFAULT 0,
    UNIQUE(account_id, api_id)
);
CREATE TABLE IF NOT EXISTS poll_state (
    account_id     INTEGER PRIMARY KEY,
    last_modified  TEXT,
    last_poll_at   INTEGER
);
CREATE TABLE IF NOT EXISTS schema_version (version INTEGER NOT NULL);
```

Implement `pub fn upsertNotification(db: *sqlite.Db, notif: types.Notification) !void`:
- INSERT OR REPLACE with ON CONFLICT(account_id, api_id) DO UPDATE SET title=excluded.title, url=excluded.url, updated_at=excluded.updated_at, is_read=CASE WHEN is_read=1 THEN 1 ELSE excluded.is_read END
- Map types: `@intFromEnum(notif.service)`, `@intFromEnum(notif.notif_type)`, `@intFromBool(notif.is_read)`
- CRITICAL: The `notif.account.token` field (if such a thing existed) must NEVER appear here. Only these fields go to SQL: id, account_id, api_id, service, notif_type, repo, title, url, updated_at, is_read

Implement `pub fn markRead(db: *sqlite.Db, notif_id: u64) !void`:
- `UPDATE notifications SET is_read=1 WHERE id=?1` with `.{ notif_id }`

Implement `pub fn markAllRead(db: *sqlite.Db, account_id: u32) !void`:
- `UPDATE notifications SET is_read=1 WHERE account_id=?1` with `.{ account_id }`

Implement `pub fn savePollState(db: *sqlite.Db, account_id: u32, last_modified: ?[]const u8, last_poll_at: i64) !void`:
- INSERT OR REPLACE INTO poll_state (account_id, last_modified, last_poll_at) VALUES (?1, ?2, ?3)

Implement `pub fn loadPollState(db: *sqlite.Db, allocator: std.mem.Allocator, account_id: u32) !?PollState`:
- SELECT last_modified, last_poll_at FROM poll_state WHERE account_id=?1
- Return null if no row

Implement `pub fn queryUnread(db: *sqlite.Db, allocator: std.mem.Allocator, account_id: ?u32) ![]types.Notification`:

- If account_id is null: SELECT * FROM notifications WHERE is_read=0 ORDER BY repo, updated_at DESC
- Allocate result slice with allocator; caller frees

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQLite DB file → store.zig | Attacker with filesystem access could read/modify the DB file |
| types.zig Account.token field | Token is in-memory; must never propagate to any serialized form |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `store.zig` — any SQL exec call | mitigate | No token field appears in any SQL INSERT/UPDATE statement; grep check in acceptance criteria enforces this |
| T-02-02 | Tampering | `store.zig` — upsertNotification | mitigate | ON CONFLICT DO UPDATE preserves `is_read=1` — a freshly-fetched API response cannot re-mark a locally-read notification as unread |
| T-02-03 | Information Disclosure | `types.zig` — Account struct | accept | token is `[]const u8` in memory; it is not `[*:0]u8` or written to any std.io writer; no log call in types.zig |
| T-02-04 | Denial of Service | `store.zig` — queryUnread | accept | DB is single-user local; no multi-tenant threat; SQLite WAL handles concurrent read/write correctly |
</threat_model>
