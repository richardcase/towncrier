Implement `pub fn startPollThread`:
```zig
pub fn startPollThread(
    allocator: std.mem.Allocator,
    account_state: *types.AccountState,
    handle: *types.TowncrierHandle,
) !void {
    const ctx = try allocator.create(PollContext);
    ctx.* = .{
        .account_state = account_state,
        .handle = handle,
        .stop_flag = std.atomic.Value(bool).init(false),
        .stop_cond = .{},
        .stop_mutex = .{},
        .http_client = http.HttpClient.init(allocator),
    };
    account_state.ctx = @ptrCast(ctx);
    account_state.thread = try std.Thread.spawn(.{}, pollThread, .{ctx});
}
```

Implement `pub fn stopPollThread`:
```zig
pub fn stopPollThread(allocator: std.mem.Allocator, account_state: *types.AccountState) void {
    if (account_state.ctx) |ctx_opaque| {
        const ctx: *PollContext = @ptrCast(@alignCast(ctx_opaque));
        ctx.stop_flag.store(true, .release);
        ctx.stop_mutex.lock();
        ctx.stop_cond.signal();
        ctx.stop_mutex.unlock();
        if (account_state.thread) |t| t.join();
        ctx.http_client.deinit();
        allocator.destroy(ctx);
        account_state.ctx = null;
        account_state.thread = null;
    }
}
```

Implement `fn pollThread(ctx: *PollContext) void` — the thread entry point:
```zig
fn pollThread(ctx: *PollContext) void {
    while (!ctx.stop_flag.load(.acquire)) {
        drainActionQueue(ctx) catch |err| {
            std.log.err("poller: action queue drain failed: {}", .{err});
        };

        doPoll(ctx) catch |err| {
            std.log.err("poller: poll failed: {}", .{err});
        };

        if (!ctx.stop_flag.load(.acquire)) {
            buildAndDeliverSnapshot(ctx) catch |err| {
                std.log.err("poller: snapshot build failed: {}", .{err});
            };
        }

        // Interruptible sleep — wakes immediately on stop signal
        ctx.stop_mutex.lock();
        defer ctx.stop_mutex.unlock();
        const interval_ns = @as(u64, ctx.account_state.account.poll_interval_secs) * std.time.ns_per_s;
        ctx.stop_cond.timedWait(&ctx.stop_mutex, interval_ns) catch {};
    }
}
```

Implement `fn drainActionQueue(ctx: *PollContext) !void`:
- Lock handle.action_mutex
- Collect all actions where account_id matches ctx.account_state.account.id into a local ArrayList
- Unlock action_mutex
- For each action: mark_read → call store.markRead(db, notif_id) + github.markRead(client, allocator, account, api_id); mark_all_read → call store.markAllRead(db, account_id)
- The allocator for temp work: use std.heap.ArenaAllocator wrapping handle.allocator, reset after each action

Implement `fn doPoll(ctx: *PollContext) !void`:
- arena allocator for this poll cycle (reset at end)
- Call github.fetchNotifications with ctx.http_client, account
- On not_modified: return immediately (no upsert, no snapshot rebuild needed)
- On success: for each notification, call store.upsertNotification; call store.savePollState with new last_modified; update account.last_modified in memory; if result.poll_interval_secs is non-null, update ctx.account_state.account.poll_interval_secs with the new value (per CORE-03 dynamic poll interval: `if (result.poll_interval_secs) |secs| { ctx.account_state.account.poll_interval_secs = secs; }`)
- On error.Unauthorized: fire on_error callback with message "GitHub authentication failed for account {id}"; set stop_flag; return

Implement `fn buildAndDeliverSnapshot(ctx: *PollContext) !void`:
- arena allocator
- Call store.queryUnread(db, handle.allocator, null) to get ALL unread across all accounts
- Sort by repo field: `std.sort.block(types.Notification, notifications, {}, compareByRepo)` where compareByRepo uses `std.mem.order(u8, a.repo, b.repo) == .lt`
- Build TowncrierSnapshot:
  - Allocate string_buf as ArrayList(u8) using handle.allocator
  - Allocate items slice of NotificationC with handle.allocator
  - For each notification: append repo to string_buf (track offset), append \x00, append title (track offset), append \x00, append url (track offset), append \x00
  - Fill NotificationC item: id, account_id, type = @intFromEnum(notif.notif_type), state = if (notif.is_read) 1 else 0, repo/title/url as pointers into string_buf
  - Allocate new TowncrierSnapshot with handle.allocator
- Acquire snapshot_lock.lockWrite()
- Free old snapshot if present: old.string_buf.deinit(); handle.allocator.free(old.items); handle.allocator.destroy(old)
- Set handle.snapshot = new_snap
- Update notif_account_map: clear map, re-add all notif.id → notif.account_id entries
- snapshot_lock.unlockWrite()
- Count unread items
- Fire callbacks (null-check first):
  ```zig
  if (ctx.handle.callbacks.on_update) |cb| {
      cb(ctx.handle.callbacks.userdata, @intCast(unread_count));
  }
  if (ctx.handle.callbacks.wakeup) |cb| {
      cb(ctx.handle.callbacks.userdata);
  }
  ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| poll thread → callbacks | Callbacks execute attacker-influenced timing (poll thread may call on_error with user-facing strings) |
| action_queue | Main thread writes; poll thread reads — concurrent access |
| snapshot swap | Poll thread writes; main thread reads via snapshot_get — RwLock boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Denial of Service | `poller.zig` — Unauthorized response | mitigate | On 401: on_error fired, stop_flag set, thread exits; does NOT retry in a loop |
| T-02-11 | Information Disclosure | `poller.zig` — on_error message | accept | Error message contains account_id (integer), not the token; formatted as "GitHub authentication failed for account {id}" |
| T-02-12 | Tampering | `poller.zig` — snapshot swap | mitigate | snapshot_lock.lockWrite() held only during pointer swap; lock not held during JSON parse, SQL query, or string buffer construction |
| T-02-13 | Elevation of Privilege | `poller.zig` — callbacks | mitigate | Callbacks are null-checked before invocation; no towncrier_* function is called from inside callback (documented in towncrier.h — deadlock if violated) |
| T-02-14 | Information Disclosure | `poller.zig` — token in memory | accept | token is held in PollContext.account_state.account.token (in-process memory); no persistence or logging; OS-level memory protection is the boundary |
</threat_model>
